PREGUNTA 6

¿Cuáles son las diferencias principales en css de los diferentes valores fixed, absolute o relative de la propiedad position? ¿Puedes poner un ejemplo práctico para que usarías cada uno de los valores comentados?

Respuesta:

Todas estas propiedades sirven para posicionar elementos. La principal diferencia entre ellos es la referencia que este tomará para su posicionamiento respecto a ella. El valor relative es el valor por defecto que toman los objetos. El valor absolute escapa al posicionamiento normal y podremos posicionarlo al elemento padre mas cercano que contenga position relative El valor fixed es como el absolute pero este siempre toma el valor de referencia de la ventana del navegador y además al hacer scroll se quedaría fijo.

El icono se posiciona arriba a la izquierda del elemento padre

    9. Pregunta Bonus: Sin dedicar más de 15 minutos, ¿Qué mejora de performance en angular propondrías para la siguiente parrilla de productos? (URL: https://www.bershka.com/es/mujer/ropa/camisetasc1010193217. html)

    Respuesta

    Lo primero sería identificar la arquitectura que podría tener la página. A simple vista se me ocurre la siguente estructura: Componentes: - Componente Wrapper - Componente Lista - Componente Item - Componente Filtro Servicios: - ProductosService El componente wrapper englobaría a todos los demás y sería el encargado de realizar la petición al servicio ProductosServices para que realice la llamada http al servidor donde tengamos los productos. Una vez hecha esta llamada asíncrona obtendriamos los datos y los tendriamos almacenados sin necesidad de hacer nuevas llamadas. Una vez obtenidos los pasariamos al componente "Lista" mediante la propiedad Input. En este componente me crearia un bucle *ngFor para ir creando componentes Item que sería cada elemento de la camiseta de forma dinámica. En el elemento lista gestionarmos la paginación así como los elementos que queramos mostrar (2,3 y 6 en este caso) (sin necesidad de hacer una nueva llamada, ya que tendremos toda la información guardada con la primera llamada.) Sería ir jugando con el JSON que recibamos para lograr nuestro resultado. El componente filtro actuaría de la misma manera que el componente List, sería el encargado de pasarle condiciones al componente Lista para que este haga una selección de los items que mostrará. Esa sería mi propuesta. Sería una buena perfomance ya que solo necesitaríamos cargar la página la primera vez para obtener toda la información que necesitamos, y despues es ir jugando con esa información a través de los componentes filtro y lista. En algunos sitios llaman al componente Wrapper componente inteligente (Seria el encargado de las llamadas asincronas) y al resto componentes Dummy (encargados de representar esa información).
